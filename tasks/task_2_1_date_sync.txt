# Task 2.1: Fix Date Sync in Battle History Dots

## Problema
Quando o usuário clica nas bolinhas de histórico de batalha, o deck atualiza corretamente, mas a **data/hora não atualiza**. O problema é que existem DOIS containers (um para cada conta) com o mesmo `id="date-1"`, e o JavaScript `getElementById` sempre retorna o PRIMEIRO elemento encontrado na página.

## Causa Raiz
- `document.getElementById('date-1')` retorna o primeiro `#date-1` encontrado (sempre da conta Principal)
- Ao clicar em bolinha da conta Secundária, a data atualizada é a da conta Principal

## Solução Necessária
Usar `element.closest('.cr-deck-card')` para encontrar o container correto, depois `container.querySelector('#date-{oppId}')` para buscar o elemento dentro desse container específico.
